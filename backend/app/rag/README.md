# rag — Семантический поиск по базе знаний

## Что делает
Гибридный поиск: dense (семантика) + sparse (BM25) → RRF fusion.
Обогащает ответы ЭС контекстом из документов (курсы, вакансии, Технопарк).

## Архитектура
```
Индексация:
  документ → чанки (500 симв.) → dense embedding + BM25 sparse → Qdrant

Поиск:
  запрос → dense + sparse → Qdrant prefetch → RRF → топ-K чанков
```

## Гибридный поиск
```
┌─────────────┐    prefetch    ┌──────────┐
│ Dense vector ├───────────────►│          │
│ (семантика)  │               │  Qdrant  │──► RRF fusion ──► результаты
│ BM25 sparse  ├───────────────►│          │
│ (ключевые)   │    prefetch    └──────────┘
└─────────────┘
```

- **Dense**: семантическое сходство (понимает смысл)
- **Sparse (BM25)**: точное совпадение ключевых слов
- **RRF**: объединяет ранги обоих поисков

## Консистентность BM25
При индексации/удалении документа BM25 корпус **полностью пересчитывается**
из актуального кэша чанков. Никакого дрифта IDF-статистик.

## API
| Метод | Путь | Роль | Описание |
|-------|------|------|----------|
| POST | `/rag/documents` | admin | Загрузка документа |
| DELETE | `/rag/documents/{source:path}` | admin | Удаление |
| POST | `/rag/search` | user | Семантический поиск |
| GET | `/rag/stats` | admin | Статистика (кол-во чанков) |

## Файлы
- `bm25.py` — BM25Encoder, sparse-векторы, кириллица
- `embedder.py` — OpenAI-совместимый /embeddings клиент
- `qdrant_client.py` — named vectors, hybrid_search, Fusion.RRF
- `service.py` — чанкинг, индексация, поиск, BM25 rebuild
- `schemas.py` — DocumentChunk, SearchRequest и др.
- `router.py` — эндпоинты
