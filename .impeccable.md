# Design Context — РС ИТО

Документ задаёт дизайн-ДНК проекта. Обновляется через `/impeccable teach`.
Все последующие дизайн-скиллы (`shape`, `craft`, `critique`, `polish` и др.) должны отталкиваться отсюда.

---

## Users

**Основная аудитория — студенты 2–6 курсов кафедры ИУ5 МГТУ им. Баумана.** Технический вуз, технический склад ума. Не боятся плотной информации, не любят маркетинговый шум, способны различать «серьёзно сделано» от «нарисовано для видимости». Используют продукт в ситуации выбора: какой курс ЦК взять, какой трек Технопарка, на чём делать фокус в курсовой. Состояние на входе — задумчивое, не спешное. Открывают скорее днём, между парами или дома за ноутом.

**Вторая аудитория — методисты/эксперты** (редактируют правила ЭС, следят за RAG-базой, смотрят трейсы запросов). Это «инженерная консоль» внутри того же продукта, но в другой тональности — плотнее, техничнее, меньше церемоний.

**Третья аудитория — комиссия защиты диплома.** Смотрит разово, должна за 15 минут увидеть, что система (а) полезна студенту, (б) прозрачна и верифицируема. Значит: студенческая часть должна читаться как зрелый продукт, а админ-консоль — как инструмент, который можно открыть рядом и доказать, что внутри детерминированные правила, а не LLM-магия.

**Job-to-be-done студента.** «Помоги понять, что мне взять / на что налечь, чтобы двигаться к [цели], учитывая что я уже прошёл и где у меня пробелы — и объясни, почему именно это.»

## Brand Personality

**Три слова: академический, спокойный, живой.**

- **Академический** — уважение к читателю, типографическая дисциплина, ясность терминов, отсутствие хайпа. Не «успей записаться, осталось 3 места». Скорее — как хорошо свёрстанный учебный материал университетского издательства.
- **Спокойный** — не давит, не торопит, не пытается продать. Много воздуха. Один фокус внимания на экран. Нет «привлекающих» цветных блоков ради вовлечения.
- **Живой** — не сухо, не бюрократично, не «университетский портал». Человеческая интонация в копирайте («ты» в дружелюбном режиме, а не «Вы, уважаемый пользователь»), тёплые нейтрали вместо стерильно-серых, одна тёплая акцентная нота в палитре.

**Эмоция на входе:** «я попал в серьёзный инструмент, ему можно доверить выбор трека, но он говорит со мной по-человечески».

**Что это НЕ:**

- НЕ типичный вузовский портал РФ (тяжёлые заголовки, перегруженность, бюрократический язык).
- НЕ типичный AI-чат (оранжево-фиолетовый gradient, glass-эффекты, «✨ powered by AI»).
- НЕ типичный SaaS-дашборд (иконка-в-квадрате над каждым заголовком, одинаковые карточки сеткой, «hero metric» с огромной цифрой).
- НЕ playful/toy — это не образовательное приложение для школьников.

## Aesthetic Direction

**Референсы:**

- **Claude (anthropic.com / claude.ai)** — тёплый off-white фон, сдержанная палитра, внимание к типографике, минимум хрома, дисциплина пустого пространства, одна тёплая акцентная нота.
- **Notion** — документный feel, чёткая иерархия через spacing и weight, ничего лишнего.
- **Традиция университетских издательств** (MIT Press, Oxford Academic, Принципы бауманского «инженерного стиля») — как метафора: хорошо свёрстанный научный текст, где форма служит содержанию.

**Антиреференсы:**

- Корпоративный SaaS в стиле Salesforce/Atlassian (плитки, синие gradient-кнопки, иконки в квадратиках).
- Типичные государственные/вузовские порталы РФ (перегруженность, жирные «официальные» тона).
- Классический «ChatGPT/Copilot» look с glass-морфизмом и неоновыми акцентами.

**Тема: только светлая.** Тёмный режим не делаем — студент и эксперт работают днём/вечером при нормальном освещении, контекст не требует dark-mode.

## Palette

Бауманский синий (≈ `#003F7F`) в оригинале — плотный и официозный. По брифу — приглушаем хрому, оставляем намёк на институциональную идентичность, но без «корпоративности». Вся палитра в OKLCH, нейтрали тинтованы к primary hue.

```css
/* Primary — бауманский navy, приглушённый */
--color-primary:           oklch(0.38 0.08 248);   /* базовый */
--color-primary-hover:     oklch(0.34 0.09 248);
--color-primary-soft:      oklch(0.94 0.02 248);   /* фон для subtle-выделений */

/* Neutrals — тинтованы к primary hue 250 */
--color-bg:                oklch(0.985 0.004 250); /* страница, тёплый off-white */
--color-surface:           oklch(1.000 0.000 0);   /* карточки/панели */
--color-surface-muted:     oklch(0.975 0.005 250);
--color-border:            oklch(0.92 0.005 250);
--color-border-strong:     oklch(0.86 0.008 250);

/* Text */
--color-text:              oklch(0.22 0.012 250);  /* основной, near-black с тёплым оттенком */
--color-text-muted:        oklch(0.50 0.010 250);
--color-text-subtle:       oklch(0.65 0.008 250);

/* Accent — одна тёплая нота для "живости" */
/* Используется редко: цель студента, highlight рекомендации, важный статус */
--color-accent:            oklch(0.58 0.13 38);    /* терракота / тёплый jasper */
--color-accent-soft:       oklch(0.95 0.03 38);

/* Status (глубокие, не кислотные) */
--color-success:           oklch(0.56 0.11 155);
--color-success-soft:      oklch(0.95 0.03 155);
--color-warning:           oklch(0.68 0.13 75);
--color-warning-soft:      oklch(0.96 0.04 75);
--color-danger:            oklch(0.50 0.15 25);
--color-danger-soft:       oklch(0.95 0.04 25);
```

Распределение по 60-30-10: 60% — bg/surface/muted нейтрали, 30% — текст и бордеры, 10% — primary (кнопки, ссылки, активные состояния) + редкие акцентные пятна. Терракота выстреливает 1–2 раза на экран максимум, иначе теряет силу.

## Typography

Обязательные требования: полная поддержка кириллицы (проект русскоязычный, дипломный), свободные лицензии, высокая читаемость длинного текста.

**Решение — три гарнитуры, каждая со своей ролью:**

| Роль | Гарнитура | Почему |
|------|-----------|--------|
| Display / заголовки / ключевые цифры | **Bitter** (slab serif) | Академический вайб без «издательского пафоса» (Playfair запрещён). Тёплый, с характером, без рыхлости Fraunces. Отличная кириллица. |
| Body / UI / формы | **Golos Text** (гротеск, Paratype) | Российская типографическая школа, спроектирован для кириллицы, тёплый гуманистический гротеск с характером. Не в списке reflex-fonts. |
| Mono / rule_id / trace / JSON | **JetBrains Mono** | Dev-tools вайб, уместный для МГТУ ИУ5 (студенты сами им пишут код). Полная кириллица. |

**Почему не дефолты.** Я рассматривал Inter + Fraunces и IBM Plex — это рефлекторные выборы, перечислены в списке бана. Bitter + Golos Text — редкая пара, сохраняет академичность без клишированности, а Golos Text родом из той же типографической культуры, что и сам проект (русский академический контекст).

**Шкала (фиксированные rem — это продуктовый UI, не маркетинг):**

```
--fs-xs:   0.75rem;   /* captions, метки */
--fs-sm:   0.875rem;  /* вторичный текст */
--fs-base: 1rem;      /* body */
--fs-md:   1.125rem;  /* акцентный body, короткие выводы */
--fs-lg:   1.375rem;  /* sub-heading */
--fs-xl:   1.75rem;   /* heading */
--fs-2xl:  2.25rem;   /* page heading */
--fs-3xl:  3rem;      /* hero (онбординг, titulo экранов) */
```

Ratio ≥ 1.25. Line-height для body 1.6, для headings 1.15. Max body width 68ch.

**Правила:**

- Заголовки — Bitter, weight 500–600. Без all-caps. Без letter-spacing на heading.
- Body — Golos Text, weight 400; акценты — 500. Без курсива в UI (курсив — только для цитат из документов RAG).
- Monospace — только для `rule_id` (`R-017`), JSON-ключей в админке, ID документов.
- Tabular numbers (`font-variant-numeric: tabular-nums`) — для оценок, семестров, процентов покрытия.

## Design Principles

1. **Объяснимость видна, но не кричит.** `Reasoning`, `rule_id`, `competency_gap`, источники RAG — всегда доступны, но в админ-режиме раскрыты, а в студенческом — за одним кликом «подробнее». Никаких всплывающих модалок для объяснений — раскрытие inline, без потери контекста.

2. **Типография как архитектура.** Иерархия через вариации гарнитур, веса и spacing'а — не через цветные блоки, плашки или подложки. Когда возникает соблазн «подложить карточку, чтобы выделить» — сначала попробовать пустым пространством и размером.

3. **Тёплый дефолт, холодный инструмент.** Студенческая сторона — тёплые нейтрали, Bitter в заголовках, человечная копия («ты»). Админская сторона — тот же stack, но плотнее, больше JetBrains Mono, меньше Bitter; не декоративная, но и не безликая.

4. **Минимум декора, максимум интенциональности.** Каждая рамка, каждая цветовая точка, каждый отступ должны быть объяснимы. Если деталь «украшает» — она лишняя. Украшением должен быть *сам текст* и *его расположение*.

5. **Уважение к читателю-инженеру.** Студенты ИУ5 разбираются в технике. Не надо прятать rule_id «чтобы не пугать» — надо показать красиво. Админам не надо наводить «дизайн» — им нужен плотный, точный инструмент.

## Tech constraints

- React 18 + Vite + TypeScript strict + Tailwind v4 + shadcn/ui (local)
- Tailwind v4 CSS-first (`@theme` в CSS, без `tailwind.config.js`)
- Фонты: Google Fonts для Bitter и JetBrains Mono; Golos Text — через `@fontsource/golos-text` (OFL) или локально
- Accessibility: WCAG AA минимум. Контрасты в палитре выше подобраны ≥ 4.5 для body, ≥ 3 для large.
- Locale: ru-RU повсеместно. Числа/даты через `Intl`, с русской локалью.

## Anti-patterns (обязательные запреты)

В дополнение к общим запретам `impeccable`, для этого проекта дополнительно:

- **Никаких «✨ AI» / «powered by» / magic wand иконок.** ЭС — не AI, это детерминированные правила. Магии не продаём.
- **Никаких gradient-заливок на тексте, кнопках, фонах.** Максимум — одноцветные solid.
- **Никаких side-stripe бордеров** на карточках рекомендаций/правил/алёртов (см. absolute bans).
- **Никаких универсальных «иконка в квадратике над заголовком»**.
- **Никаких sparkline-декораций** в карточках рекомендаций. Радар компетенций — да (он информативен). Мини-графики без смысла — нет.
- **Никаких «card-in-card»**. Карточка рекомендации внутри карточки чата — плоско, без вложенного фона.
